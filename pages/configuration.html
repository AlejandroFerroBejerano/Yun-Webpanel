¿Por qué no usar BBDD?

Hay que destacar que el Arduino-Yún al no tener muchas entradas y ser sus recursos tanto de memoria como de procesamiento limitados, resulta ineficiente y complicado la intalación y usos de sistemas gestores de bases de datos. Además este proyecto está pensado para que cualquier persona solo con copiar en la estructura de directorios los archivos del proyecto y habiendo cargado el código de ejemplo de la librería bridge puedan hacer uso del mismo sin mayor contratiempo.

¿Cómo lograr persistencia?

Para lograr tener persistencia de datos se ha decidido usar el método más sencillo y factible, ficheros.

¿Cómo editar y cambiar nuestra configuración?

1. Acceder por ssh a la estructura de directorios donde se encuentra el proyecto

2. Abrir el archivo sj/script.js a partir de la línea 40 a la 48

3. Editarlas entradas como se muestra a continuación:

{idButton:'13', state:false ,mode:'output' ,type:'digital' ,description:'Comedor'},
